How Documetro Works

Documetro builds a temporary hybrid retrieval index from your uploaded files, then answers by combining classical search signals, dense embeddings, and provider-assisted reranking.

This project has been developed by Yunus Emre Vurgun (yunusemrevurgun.com)

Embeddings

  • Text corpus: nvidia/llama-3.2-nv-embedqa-1b-v2:free
  • Visual corpus: nvidia/llama-nemotron-embed-vl-1b-v2:free
  • Strategy: One embedding space per corpus, selected by modality.

Providers

  • Embedding provider: OpenRouter
  • Multimodal extraction model: openai/gpt-4o-mini
  • Answer drafting model: liquid/lfm2-8b-a1b
  • Reasoning + reranking: Nous Hermes / Hermes-4-70B

Pipeline Summary

  1. Uploads are parsed locally into sections and chunks.
  2. Images, audio, and video are converted into retrieval-ready text before indexing, using the multimodal extraction model when OpenRouter is enabled.
  3. Retrieval blends TF-IDF, latent semantic scoring, and embeddings, with embeddings as the main signal.
  4. Nous Hermes can refine the query and rerank evidence before answer generation.
  5. The final answer is grounded in the highest-scoring excerpts from your uploaded files.
